Product & Job API

Vollständige REST-API auf Basis von Django & Django REST Framework

Django 5 Django REST Framework JWT Auth SQLite / PostgreSQL

Datenmodelle

🗂️

Category

Produktkategorien als Fremdschlüssel-Basis. Jede Kategorie kann beliebig viele Produkte enthalten (related_name='products').

📦

Product

Kern-Entity mit Name, Beschreibung, Preis, Lagerbestand, SKU, Bild und Status. DB-Indizes auf häufig gefilterten Feldern, Schutz-FK zur Kategorie.

💼

Jobs

Stellenanzeigen mit Name, Beschreibung und Gehalt. Sortierung standardmäßig nach Gehalt absteigend. CRUD nur für authentifizierte Nutzer.

API-Endpunkte

Method URL Beschreibung
GET /api/categories/ Alle Kategorien (paginiert)
POST /api/categories/ Neue Kategorie erstellen
GET /api/products/ Produkte — Filter, Suche, Sortierung
POST /api/products/ Neues Produkt anlegen
GET /api/products/<id>/ Einzelnes Produkt abrufen
PATCH /api/products/<id>/ Produkt teilweise aktualisieren
DELETE /api/products/<id>/ Produkt löschen
GET /api/jobs/ 🔒 Alle Jobs — Filter & Suche
POST /api/jobs/ 🔒 Neuen Job erstellen
GET /api/jobs/<id>/ 🔒 Einzelnen Job abrufen
PUT PATCH /api/jobs/<id>/ 🔒 Job aktualisieren
DELETE /api/jobs/<id>/ 🔒 Job löschen
GET /api/app-manager/ 🔒 App Manager (diese Seite)

🔒 = Authentifizierung erforderlich (Bearer Token)

Features

PaginierungEigene Pagination-Klassen für Categories, Products und Jobs
FilterungExakte Filter per URL-Parameter, z.B. ?active=true
VolltextsucheGlobale Suche über Name, Beschreibung, SKU und Kategorie
SortierungFlexibles Ordering per ?ordering=price oder -salary
Queryset-Optimierungselect_related für JOIN-basiertes Laden der FK-Daten
DB-IndizesIndizes auf SKU, active, category und name für schnelle Abfragen
JWT AuthenticationSchützt Job-CRUD und den App Manager
BilduploadProduktbilder via ImageField unter /media/products/
